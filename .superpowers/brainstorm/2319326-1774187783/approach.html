迁移方案选择

选择 React 重构的实施路线

A

原地替换（推荐）

删除 frontend/ 内的 Vue 代码，在原目录创建 React 项目。build.sh 和 nginx 配置无需改动，dist 产物路径不变。

优点

  • 构建/部署零改动
  • 路径一致性好

缺点

  • 无法快速回滚到 Vue
B

新目录 + 交换

创建 frontend-react/，Vue 代码保留在 frontend/。新前端验证稳定后，更新 build.sh 指向新目录。

优点

  • 旧代码保留，随时回滚
  • 可并行对比

缺点

  • 需要改 build.sh / nginx 路径
  • 目录稍显冗余

两种方案共同的技术栈

核心框架
React 18 + Vite + React Router 7
UI 系统
Tailwind CSS 4 + shadcn/ui（直接复用 assert/ 组件）
状态管理
React Context（Auth）+ 组件 State（Chat/SSE）
