# 口播稿改写系统 — 架构设计文档

**日期**: 2026-03-21
**状态**: 已确认
**版本**: v1.0

---

## 1. 系统概述

基于 Claude Code Skill + 远端 Go 服务的多用户口播稿改写平台。用户通过 Claude Code 触发创作流程，Skill 调用远端 API 获取用户风格档案、热点资讯、素材库，Claude LLM 负责全部智能分析与生成，远端服务负责持久化存储与管理后台。

### 核心目标

- 多用户隔离：每个用户有独立的风格档案、稿件历史
- 智能创作：5角色分析流程 + 相似度 < 30% 质量门控
- 离线可用：本地文件缓存层支持远端不可达时降级运行
- 管理可视：超管 + 用户自助的 Web 管理后台

## 3. 本地 vs 远端职责拆分

### 3.1 Skill 本地职责（LLM 智能层）

| 功能 | 实现方式 | 数据来源 |
|------|----------|----------|
| 文本提取 + 纠错 | Claude 直接处理 | 用户输入 |
| 爆款 DNA 拆解（5角色分析） | Claude prompt 编排 | 用户输入 |
| 素材补齐 | Claude WebSearch + 远端素材库 | 远端/本地缓存 |
| 风格注入 | Claude 读取风格档案后生成 | 远端/本地缓存 |
| 终稿撰写 | Claude 生成 | — |
| 相似度评估 | Claude LLM 打分（4维度加权） | — |
| 爆款评分 | Claude LLM 打分 | — |
| 用户确认交互 | skill 流程控制 | — |

### 3.2 远端服务职责（数据 + 资讯层）

| 模块 | 具体职责 | 接口 |
|------|----------|------|
| 用户档案 | 注册/登录、基础信息 | `GET/PUT /user/profile` |
| 风格档案 | 存储风格向量、历史偏好、口头禅库 | `GET/PUT /user/style` |
| 稿件管理 | 存储历史稿件，支持搜索/标签/收藏 | `GET/POST /scripts` |
| 素材库 | 按主题查询金句、案例、数据 | `GET /materials?topic=XX` |
| 热点雷达 | 定时抓取各平台热榜，结构化存储 | `GET /hotspot?platform=XX` |
| 离线同步 | 接收本地离线队列批量上传 | `POST /sync` |
| Admin Backend | Web 管理后台 | Web UI |

## 4. 本地文件缓存设计

### 4.1 目录结构

```
~/.skill-imm/
├── config.json          # 远端 API 地址、用户 token
├── profile.json         # 用户档案缓存（TTL: 7天）
├── style.json           # 风格档案缓存（TTL: 7天）
├── hotspot.json         # 热点数据缓存（TTL: 1小时）
├── materials/
│   ├── finance.json     # 按主题分类的素材缓存
│   └── lifestyle.json
├── scripts/
│   ├── 2026-03-21-xxxx.md  # 本地稿件副本（永久保留）
│   └── index.json          # 稿件索引
└── sync_queue.json      # 离线时待上传的操作队列
```

### 4.2 缓存 TTL 策略

| 数据类型 | TTL | 原因 |
|----------|-----|------|
| 热点数据 | 1 小时 | 时效性强 |
| 素材库 | 24 小时 | 更新不频繁 |
| 风格档案 | 7 天 | 变化缓慢 |
| 用户档案 | 7 天 | 基本稳定 |
| 稿件文件 | 永久 | 本地副本，不过期 |

### 4.3 降级策略

```
正常模式:
  读: 检查本地缓存 TTL → 过期则请求远端 → 更新本地缓存
  写: 先写远端 → 成功后同步本地缓存

降级模式（远端不可达，ping 超时 2s）:
  读: 直接读本地缓存（忽略 TTL 过期，输出降级提示）
  写: 写本地文件 + 追加到 sync_queue.json

恢复模式（重新连接远端）:
  启动检测远端可达性
  → 读取 sync_queue.json
  → 按序上传（幂等操作）
  → 清空队列，打印同步结果
```

## 5. API 接口设计

### 5.1 认证

本地 `~/.skill-imm/config.json` 存储配置：

```json
{
  "api_base": "https://your-server.com/api/v1",
  "token": "eyJhbGci...",
  "user_id": "u_xxxx",
  "timeout_ms": 5000
}
```

所有请求携带 `Authorization: Bearer <token>`。

### 5.2 接口清单

```
# 健康检查
GET  /ping

# 用户 & 风格
GET  /user/profile
PUT  /user/style

# 稿件
GET  /scripts               ?page=1&limit=20
GET  /scripts/:id
POST /scripts
PUT  /scripts/:id/tags

# 素材 & 热点
GET  /materials             ?topic=XX&limit=10
GET  /hotspot               ?platform=douyin|xiaohongshu|weibo

# 离线同步
POST /sync
```

### 5.3 Skill 调用流程

```
创作开始
  ↓
1. GET /ping（超时 2s）→ 判断正常/降级模式
2. 拉取风格档案（缓存优先）
3. 拉取热点资讯（缓存优先，TTL 1h）
4. 查询相关素材（GET /materials?topic=XX）
5. Claude 执行 5角色分析 + 生成终稿
6. 保存稿件
   ├─ 正常 → POST /scripts → 同步写本地
   └─ 降级 → 写本地 + 追加 sync_queue.json
7. 静默更新风格档案（PUT /user/style，失败不阻断）
```

## 6. Admin Backend 功能设计

### 6.1 超管视角（role: admin）

| 模块 | 功能 |
|------|------|
| 用户管理 | 用户列表、账号启用/禁用、重置 token、查看某用户稿件 |
| 全量稿件 | 所有稿件列表、按用户/日期/平台/评分筛选、导出 CSV |
| 热点配置 | 配置抓取数据源（平台、关键词、频率）、查看抓取日志 |
| 素材库管理 | 新增/编辑/删除公共素材，打标签分类 |
| 系统配置 | TTL 参数、同步策略、API 限流配置 |
| 数据看板 | 总生成量、用户活跃度、平均爆款评分趋势、热门选题分布 |

### 6.2 用户自助视角（role: user）

| 模块 | 功能 |
|------|------|
| 我的稿件 | 历史稿件列表、搜索、查看详情（含质量报告）、打标签、导出 |
| 风格档案 | 查看当前风格画像、手动编辑修正、查看进化历史 |
| 素材收藏 | 从公共素材库收藏、新增私有素材、按主题管理 |

## 9. 质量保证

| 约束 | 标准 |
|------|------|
| 相似度 | < 30%（4维度加权：词汇30% + 句式25% + 结构25% + 观点20%） |
| 最大重写次数 | 3次，超过后提示用户调整方向 |
| 降级透明度 | 降级模式下明确提示用户，标记缓存数据的过期时间 |
| 同步幂等性 | sync_queue 中的操作必须设计为幂等，避免重复上传 |
